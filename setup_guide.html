1

Create Your Telegram Bot

Open Telegram and message @BotFather. Send these commands:

# Talk to @BotFather on Telegram
/newbot
# BotFather asks for a name:
API Finder Bot
# Then asks for a username (must end in "bot"):
my_api_finder_bot
# BotFather gives you a token like:
1234567890:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw

💾 Save that token — you'll need it in step 3.

2

Install Dependencies

Only one external library needed:

# Navigate to the bot directory
cd api_finder_bot

# Install dependencies (just python-telegram-bot==20.7)
pip install -r requirements.txt

Works with Python 3.10+ — No database, no Redis, no Docker needed.

3

Configure Your Token

Set your bot token as an environment variable:

# Option A: Export in terminal (Linux/macOS)
export TELEGRAM_BOT_TOKEN=1234567890:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw

# Option B: Create a .env file
cp .env.example .env
# Then edit .env and add your token

# Option C: Windows PowerShell
$env:TELEGRAM_BOT_TOKEN="your_token_here"
4

Run The Bot

python bot.py

# You should see:
INFO | 📥 Fetching data from GitHub...
INFO | 📊 Parsed 1430 API entries
INFO | 🗂 Built indexes: 45 categories, 8200 keywords
INFO | ✅ Indexed 1430 APIs
INFO | 🚀 Bot is live! Polling for messages...
First run: The bot fetches the public-apis README from GitHub (~400KB). This takes 1–3 seconds. Every search after that is instant — the data lives in RAM.

Deploy to Production

Run the bot 24/7 on a free server:

# Option A: Railway.app (easiest — free tier)
# Push to GitHub → Connect repo → Set env var → Deploy

# Option B: Run forever with nohup
nohup python bot.py > bot.log 2>&1 &

# Option C: systemd service (Linux VPS)
sudo systemctl enable api-finder-bot
sudo systemctl start api-finder-bot

# Option D: Docker
docker run -e TELEGRAM_BOT_TOKEN=xxx -d api-finder-bot

💡 For Railway/Render: just set TELEGRAM_BOT_TOKEN as an environment variable in their dashboard.
